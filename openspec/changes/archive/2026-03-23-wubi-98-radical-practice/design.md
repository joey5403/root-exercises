## Context

五笔 98 输入法共有 200+ 个字根，分布在 25 个键位上。学习者需要反复练习才能形成肌肉记忆。目前"字根"文件夹中已包含按区域组织的 JPG 图片（键名字、撇区、繁体字根等），但缺乏互动练习工具。

目标用户：五笔 98 初学者和需要复习的进阶用户。

## Goals / Non-Goals

**Goals:**
- 单 HTML 文件实现，无需构建工具
- 随机显示字根图片，用户输入对应键位
- 本地存储错误次数、练习历史
- 实时反馈（正确/错误）和统计面板
- 支持全部练习和错题复习模式

**Non-Goals:**
- 不需要后端服务
- 不需要用户登录系统
- 不支持多设备同步
- 不包含完整五笔编码练习（仅字根）

## Decisions

### 1. 技术栈：纯 HTML + CSS + Vanilla JavaScript

**理由:**
- 零依赖，无需 npm/打包工具
- 浏览器原生支持 localStorage
- 文件小巧（<100KB），加载快
- 易于理解和修改

**替代方案:** React/Vue
**排除原因:** 过度复杂，需要构建流程，不适合简单练习工具

### 2. 字根数据结构：JavaScript 对象映射

```javascript
const radicals = [
  { key: 'g', image: '字根/键名字/g3.jpg', name: '王' },
  { key: 'f', image: '字根/键名字/f2.jpg', name: '土' },
  // ...
];
```

**理由:**
- 简单直接，易于维护
- 可轻松扩展属性（如错误次数）
- 与图片文件路径直接对应

### 3. 本地存储结构：localStorage

```javascript
{
  "wubi98_stats": {
    "g": { errors: 3, correct: 10, lastPracticed: timestamp },
    "f": { errors: 5, correct: 8, lastPracticed: timestamp },
    // ...
    "totalPractices": 100,
    "sessionStartTime": timestamp
  }
}
```

**理由:**
- 浏览器原生支持
- 持久化存储，关闭浏览器不丢失
- 简单易用，无需额外库

### 4. 图片加载策略：动态创建 Image 对象

**理由:**
- 避免页面加载时一次性加载所有图片
- 按需加载，节省内存
- 支持预加载下一张图片

### 5. 练习模式：两种模式

1. **全部练习**: 随机显示所有字根
2. **错题复习**: 只显示错误次数>0 的字根

**理由:**
- 满足不同学习阶段需求
- 错题本功能提高复习效率

## Risks / Trade-offs

### [风险] 图片路径硬编码

**问题:** 图片路径写在代码中，移动文件会失效

**缓解:** 
- 在 README 中说明目录结构
- 提供图片映射配置文件（可选）

### [风险] localStorage 容量限制

**问题:** localStorage 有 5MB 限制

**缓解:** 
- 仅存储统计数据（<1KB）
- 不存储图片数据

### [风险] 浏览器兼容性

**问题:** 老旧浏览器可能不支持 localStorage

**缓解:** 
- 降级处理：不支持时提示用户
- 目标浏览器：现代浏览器（Chrome/Firefox/Edge）

### [Trade-off] 无云端同步

**问题:** 数据仅保存在本地，更换设备丢失

**缓解:** 
- 提供导出/导入功能（后续版本）
- 保持简单性优先

## Migration Plan

1. 创建 `index.html` - 主页面
2. 创建 `style.css` - 样式文件
3. 创建 `app.js` - 核心逻辑
4. 创建 `data.js` - 字根数据映射
5. 测试所有功能
6. 验证 localStorage 持久化

**无需数据库迁移，无需后端部署**
